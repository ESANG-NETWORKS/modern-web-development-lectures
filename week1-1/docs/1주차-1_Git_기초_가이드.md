# 1주차-1 Git 기초 가이드

## 목표

- VS Code Source Control 화면에서 변경 파일 상태를 읽는다.
- Stage, Unstage, Commit 흐름을 직접 실행한다.
- 브랜치를 만들고 원격으로 Push 하는 기본 흐름을 경험한다.
- 간단한 충돌을 Merge Editor로 해결한다.

## 권장 시간

- 총 60분
- 0~10분: Git이 필요한 이유와 오늘 목표 설명
- 10~25분: Source Control 기본 조작
- 25~45분: 브랜치와 원격 실습
- 45~55분: 충돌 미니 실습
- 55~60분: 정리와 과제 안내

## 실습 체크리스트

### 1. 변경 사항 확인

- 수정 파일 상태 확인
- Stage / Unstage 실습
- 커밋 메시지 작성

### 2. 브랜치 작업

- `develop` 또는 기준 브랜치 최신화
- `feature/git-basic-{이름}` 브랜치 생성
- 문서 한 파일 수정 후 커밋
- 원격 Push 및 Upstream 연결

### 3. 충돌 해결

- 같은 파일 같은 줄을 두 브랜치에서 수정
- VS Code Merge Editor에서 해결
- 해결 후 커밋 생성

## 과제

- 각자 실습 브랜치에서 문서 1회 추가 수정
- 다음 회차 전 원격 Push 완료
