# ESSecrets Usecase 반영 지시서

본 문서는 `docs/UML.drawio`의 Usecase를 구현 지시 기준으로 정제한 문서입니다.  
목표는 **권한 경계 명확화**, **요청/승인 흐름 명시**, **화면/백엔드 수용 기준 통일**입니다.

용어 기준은 [도메인/키워드 가이드](DOMAIN_KEYWORD_GUIDE.md)를 우선 적용합니다.

## 1) 적용 범위

- 대상 화면: Login, Main (시크릿 목록 조회/원문 조회/추가/수정/삭제요청/접근기록)
- 대상 액터: 팀원, 팀장
- 대상 기능: 인증, 시크릿 관리, 권한 관리, 접근기록

## 2) 액터 정의

### 팀원

- 로그인/로그아웃
- 정보 조회
- 정보 추가
- 정보 변경
- 정보 삭제요청
- 정보 접근기록 조회(권한 허용 범위 내)

### 팀장

- 팀원의 모든 권한 포함
- 정보 삭제 승인 및 실제 삭제 수행
- 정보 접근권한 부여/제거
- 직원 계정 접근 비활성화/활성화
- 직원 목록 조회

## 3) 유스케이스 관계 표준 (필수)

기존 다이어그램의 점선 관계는 아래 규칙으로 해석/정리한다.

- `시크릿 목록 조회` **include** `접근기록 생성`
  - 조회가 성공하면 접근기록(AuditLog)은 자동 생성되어야 한다.
- `정보삭제` **include** `삭제승인 확인`
  - 팀장은 승인된 요청 건에 대해서만 삭제 가능하다.
- `정보삭제 요청`은 팀원의 독립 유스케이스
  - 요청 생성 후 상태값(`Pending/Approved/Rejected`)으로 관리한다.
- `민감정보 원문 조회` **include** `비밀번호 재확인`
  - 세션 유효 중에도 원문 확인 시 재인증을 요구한다.

## 4) 용어 분리 (중복 제거)

혼동 방지를 위해 권한 관련 용어를 다음으로 통일한다.

- **직원 계정 권한 관리**: 직원 단위 활성/비활성, 역할(팀원/팀장) 변경
- **정보 접근 권한 관리**: 특정 정보 레코드에 대한 조회/수정/관리 권한 부여·회수

## 6) 화면 반영 지시

## 6.1 Login

- 필수 상태 정의
  - 기본, 입력 오류, 인증 실패, 로딩, 버튼 비활성
- 입력 정책
  - Enter 제출 허용
  - 연속 실패 임계치 초과 시 일시 잠금
- 메시지 규칙
  - 계정 존재 여부 노출 금지(동일 오류 문구 사용)

## 6.2 Main

- 목록 컬럼 정책
  - ID/Password 원문 상시 노출 금지
  - 기본은 마스킹/숨김, `조회` 액션 + 재인증 후 제한 시간 노출
- 행 액션 정책
  - 팀원: 수정/삭제요청/접근기록
  - 팀장: 수정/삭제(승인된 건)/접근기록/정보 접근권한 관리
- 상태 화면
  - 로딩, 검색결과 없음, 권한 없음, 서버 오류, 세션 만료
- 파괴적 액션
  - 삭제/권한회수/비활성화는 확인 모달 + 사유 입력 필수

## 7) 접근기록(AuditLog) 최소 스키마

필수 항목:

- `timestamp`
- `actorUserId`
- `actorRole`
- `actionType` (LIST_VIEW, DETAIL_VIEW, ADD, UPDATE, DELETE_REQUEST, DELETE_APPROVE, DELETE_EXECUTE, GRANT, REVOKE, LOGIN, LOGOUT)
- `targetType` / `targetId`
- `result` (SUCCESS/FAIL)
- `ip`
- `userAgent`
- `reason` (해당 시)

## 8) API/백엔드 지시

- 서버는 UI 표시 여부와 무관하게 권한을 최종 검증한다.
- 모든 변경/조회 핵심 이벤트는 트랜잭션 경계 내에서 로그를 남긴다.
- 삭제는 Soft Delete 기본, 물리 삭제는 별도 운영 정책 승인 후 수행한다.
- 동시성 충돌(수정/삭제 경합) 시 표준 에러코드로 응답한다.

## 9) 수용 기준 (Acceptance Criteria)

- 팀원이 정보 삭제를 직접 실행하면 403 반환, 삭제요청 생성은 성공한다.
- 팀장이 승인하지 않은 요청 건은 삭제 API가 실패한다.
- 정보 상세 조회 시 세션 재인증을 통과하지 못하면 원문이 반환되지 않는다.
- 시크릿 목록 조회 성공 건마다 접근기록이 1건 이상 생성된다.
- 로그인 실패 메시지는 계정 존재 여부를 유추할 수 없어야 한다.

## 10) 우선순위

### P1 (즉시)

- 권한 매트릭스 서버 강제
- 민감정보 노출 정책(마스킹 + 재인증)
- 삭제요청/승인/실행 3단계 분리
- 접근기록 자동 생성

### P2

- 직원 계정 관리(비활성화/활성화)
- 접근기록 고급 검색(기간/행위/대상)

### P3

- 운영 편의 기능(알림, 배치 정리, 리포트)
