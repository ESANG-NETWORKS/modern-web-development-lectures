# 1주차-2 GitHub Flow와 PR 가이드

## 목표

- GitHub Flow 기준으로 브랜치 생성부터 PR 병합까지 한 사이클을 수행한다.
- 리뷰 코멘트를 반영하고 병합 전 확인 포인트를 익힌다.
- 팀 공용 산출물을 공유하고 피드백을 정리한다.

## 권장 시간

- 총 60분
- 0~10분: GitHub Flow 규칙 설명
- 10~30분: PR 생성과 리뷰 실습
- 30~45분: 팀 산출물 공유
- 45~55분: Copilot 활용 미니 소개
- 55~60분: 회고와 다음 주 연결

## 실습 체크리스트

### 1. PR 생성

- `feature/flow-pr-{이름}` 브랜치 생성
- 작은 변경 1건 커밋 및 Push
- 제목과 본문을 포함한 PR 생성

### 2. 리뷰 반영

- 리뷰 코멘트 작성 또는 수신
- 수정 후 추가 커밋 반영
- 병합 전 변경 범위 재확인

### 3. 병합과 정리

- Squash Merge 또는 Rebase Merge 선택
- 병합 후 브랜치 정리
- 다음 작업 단위를 위한 브랜치 전략 점검
